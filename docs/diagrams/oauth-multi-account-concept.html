msgvault · oauth apps

Default and Named OAuth Apps

Read each lane left to right: an OAuth app authorizes one or more add-account runs, and every run writes its own token file. Each example account keeps one color, from its command to the token it creates. The default app covers personal Gmail; a named app covers a Workspace org that restricts OAuth to its own apps.

OAuth app authorizes — add-account creates — ~/.msgvault/tokens/
Default app
Personal Gmail. One client_secret.json covers them all.
[oauth]
client_secrets = "client_secret.json"
→
msgvault add-account personal@gmail.com
msgvault add-account other@gmail.com
→
personal@gmail.com.json
other@gmail.com.json
OAuth app authorizes — add-account creates — ~/.msgvault/tokens/
Named app
A Workspace org that allows only its own OAuth app.
[oauth.apps.acme]
client_secrets = "acme_secret.json"
→
msgvault add-account you@acme.com --oauth-app acme
→
you@acme.com.json
Each account gets its own token file. sync, serve, and re-auth resolve the right credentials automatically.
